## 1) 프로젝트 개요

본 프로젝트는 **팀 프로젝트 협업 지원 시스템**으로 
조별과제 및 팀 프로젝트에서 자주 발생하는 **기여도 및 활동 참여의 불투명성 문제**를 줄이기 위해 개발되었다.  
평가자가 참여자별 기여도를 **객관적으로 비교·평가**할 수 있도록 돕는 **평가 보조 웹 프로그램**이다.

참여자는 자신이 수행한 활동을 **활동 로그 입력 양식**을 통해 기록하며  
누적된 활동 내역은 평가 관점에 맞게 **자동으로 집계·시각화**된다.  
이 결과는 **평가용 결과 보고서(PDF)** 형태로 정리되어  
프로젝트 전반의 진행 과정과 개인별 기여도를 한눈에 확인할 수 있도록 구성되어 있다.

---

### 1.1 개발 배경 및 문제 정의

기존의 조별 프로젝트 평가는 다음과 같은 한계를 가진다.

- 역할 분담에서 갈등이 일어남 
- 팀원 간 기여도 차이가 발생함  
- 결과물 중심 평가로 인해 평가의 공정성이 낮아질 수 있음  

대부분의 팀 프로젝트에서는  
“**누가 무엇을 얼마나 수행했는지**”에 대한 기록이 체계적으로 남지 않는다.  
이로 인해 최종 결과물만을 기준으로 평가가 이루어지고  
팀원 간 기여도 인식 차이나 평가의 불공정성에 대한 문제가 발생하기도 한다.

또한 프로젝트 진행 중 활동 내역이 메신저(카카오톡) 메모 등으로 흩어져 있어  
전체 진행 흐름을 한눈에 파악하기 어렵다는 점도 한계로 작용한다.

---

### 1.2 프로젝트 목적

이러한 문제를 해결하기 위해, 본 프로젝트는  
**활동 로그 기반 기여도 관리 및 시각화 시스템**을 구현하는 것을 목표로 한다.

활동 기록을 표준화된 로그 형태로 저장하고  
기록된 데이터를 바탕으로 기여도를 자동으로 집계함으로써  
평가자가 판단에 필요한 핵심 정보를 **직관적으로 확인**할 수 있도록 돕는다.

---

### 1.3 개발 목표

본 프로젝트의 주요 개발 목표는 다음과 같다.

- 팀원의 모든 활동을 **로그 단위로 기록**
- 기록된 데이터를 기반으로 **기여도 및 활동 흐름 자동 집계**
- 팀원 정보를 활용한 **역할 추천 시스템 제공**
- **익명 평가 기능**을 통한 활동 품질 피드백 수집
- **별도의 서버 없이 동작하는 경량 웹 시스템 구현**

---

### 1.4 초기 계획 대비 변경 사항

- **역할 분담 추천 방식 변경**  
  초기에는 MBTI, 성격 유형, 전공 등을 활용한 역할 추천 시스템을 고려하였다.  
  그러나 해당 요소들이 객관적인 평가 기준으로 활용되기 어렵다고 판단하여  
  최종적으로는 학과 정보, 선호 역할 3개, 비선호 역할 1개를 입력받는  
  **단순 규칙 기반 역할 추천 방식**으로 변경하였다.

- **일정 조율 기능 제외**  
  팀원별 가능·불가능 시간대를 기반으로 공통 시간을 도출하는 기능을 계획하였으나  
  해당 기능은 채팅형 시스템에서 더 효과적으로 활용될 수 있다고 판단하여  
  본 시스템의 구현 범위에서 제외하였다.

- **개발 대상 초점 변경**  
  초기에는 학생 사용자 중심으로 설계하였으나  
  개발을 진행하면서 본 시스템의 핵심 가치가  
  **평가자가 팀원 개별 기여도를 공정하게 판단할 수 있도록 지원하는 것**에 있음을 인식하게 되었다.  
  이에 따라 평가자 관점의 정보 제공에 초점을 맞추어 개발을 진행하였다.

- **AI 기능 적용 제외**  
  초기 기획 단계에서는 AI 기반 요약 및 기여도 분석을 고려하였다.  
  그러나 외부 AI API 연동 시 발생하는 비용을 예산 처리하기에 어려움이 있으므로 
  최종 구현 단계에서는 **규칙 기반 자동 집계 방식**을 적용하였다.

### 1.5 데모 데이터 사용 방법

본 프로그램의 기능 시연 및 평가를 위해 **데모 데이터**를 제공한다.

1. 상단의 **“ 샘플 데이터 로드”** 버튼 클릭  
2. 팀원 관리 화면에서 **역할 추천 보기**를 통해 추천 역할 확인 및 선택  
   -  추천 역할을 선택해야 타임라인 및 평가 데이터가 정상적으로 로드된다.
3. **“요약 & PDF 생성”** 버튼 클릭 → 기여도 요약 대시보드가 아래 페이지로 생성.

데모 데이터 기준으로 다음과 같은 정보가 자동으로 생성된다.

- 전체 팀원 기여도 요약
- 주차별 활동 히트맵
- 팀원별 활동 유형 비중
- 팀원별 기여도 상세 정보

PDF 생성 기능을 실행하면  
프로젝트 전반의 활동 내역과 개인별 기여도를 확인할 수 있는  
**평가용 결과 보고서(PDF)** 가 생성된다.

 데모 데이터는 시연 목적의 데이터이므로  
 활동 상세 보기나 증빙 자료 열기 시 실제 외부 페이지가 출력되지 않을 수 있다.

## ⚠️ 실행 중 발생할 수 있는 문제 해결

### 사이트 접속 시 404 오류가 발생하는 경우

GitHub Pages 특성상 최초 배포 이후 또는 캐시 상태에 따라  
사이트 접속 시 404 오류가 발생할 수 있다.  
아래 방법을 순서대로 시도하면 대부분 해결된다.

1. 페이지 **새로고침(F5)** 실행
2. **시크릿(Incognito) 모드**에서 접속
3. 주소창 URL **맨 뒤에 `/` 추가**  
   - 예: `https://chaerin-yu1203.github.io/-TeamFlow-/`

---

### PDF 파일이 생성되지 않는 경우

PDF 생성 기능은 **브라우저 인쇄 기능**을 사용한다.  
아래 사항을 확인한다.

- Chrome 또는 Edge 브라우저 사용 권장
- 팝업 차단 해제
- 인쇄 창에서 **대상: “PDF로 저장”** 선택


> PDF 생성 버튼 클릭 시 브라우저 인쇄 창이 열리지 않는 경우,  
> 팝업 차단 여부를 먼저 확인한다.

---

### 데모 데이터 로드 후 화면이 정상적으로 보이지 않는 경우

- **역할 추천을 선택하지 않은 상태**에서는  
  일부 타임라인 및 평가 데이터가 표시되지 않을 수 있다.
- 팀원 관리 화면에서 **역할 추천 보기 → 역할 선택**을 먼저 진행한다.

### 1.6 기대 효과 및 향후 발전 방향

본 시스템을 통해 팀 프로젝트에서 자주 발생하는 **기여도 논란을 줄이고**, 
평가자의 주관적인 판단 부담을 완화할 수 있다.  
활동 기반의 객관적인 근거를 제공함으로써  
프로젝트 진행 과정을 보다 명확하게 파악할 수 있다.

향후에는 서버 기반 아키텍처로 확장하여  
사용자 계정 관리, 프로젝트 이력 관리, 다중 프로젝트 운영 기능을 추가할 수 있다.  
또한 평가 목적(과제 제출, 성과 평가, 회고 등)에 따라  
다양한 형식의 결과 보고서를 자동 생성하는 방향으로 확장할 수 있다.

본 시스템은 대학 팀 프로젝트뿐만 아니라  
연구 과제, 협력 프로젝트, 소규모 조직의 협업 평가 등  
다양한 환경으로 확장 적용될 수 있다.

## 2) 핵심 기능

### ✅ 팀원 / 역할 관리 및 역할 추천
- 팀원 정보(학과, 선호 역할 1~3순위, 비선호 역할 1개)를 입력받아 규칙 기반으로 역할 추천
- 역할 충돌을 줄이고 팀 합의를 통해 최종 역할 확정

### ✅ 활동 로그 입력 및 타임라인
- 활동을 “무엇을 / 왜 / 어떻게” 관점으로 기록
- 활동 카테고리, 참여자, 점수, 날짜·시간, 증빙 링크 등 저장
- 시간순 타임라인을 통해 프로젝트 진행 흐름 확인

### ✅ 익명 평가(피드백)
- 특정 활동 로그에 대해 익명 한줄 피드백과 점수(예: 1~5) 입력
- 비속어 필터링(블러 처리), **연속 제출 제한(10분 쿨타임)** 적용

### ✅ 기여도 요약 대시보드
- 전체 활동 요약
- 주차·일자 기반 활동 히트맵
- 팀원별 활동 유형 비중
- 팀원별 기여도 상세 정보 제공

### ✅ PDF 평가 리포트 생성
- 대시보드 결과를 평가자 관점의 레이아웃으로 자동 정리
- 프로젝트 정보, 역할 분담, 히트맵, 활동 그래프, 멤버별 요약, 익명 평가, 전체 타임라인 포함

## 3) 기술 스택

- **Frontend**: HTML, CSS, Vanilla JavaScript (ES6+)

## 4) 동작 방식 (아키텍처 요약)

### 데이터 저장 구조 (localStorage)

`teamContributionApp` 키에 아래와 같은 형태로 데이터가 저장된다.

- `project`: 프로젝트 기본 정보
- `members`: 팀원 목록
- `logs`: 활동 로그 목록 (단일 기록 소스)
- `summaries`: 자동 집계 결과(대시보드)
- `feedbacks`: 익명 평가 데이터

모든 사용자 입력 및 수정 사항은 `autoSave()`를 통해 즉시 저장된다.

### 코드 실행 흐름

1. 페이지 로드 → `state.js`에서 localStorage 데이터 로딩  
2. `app.js`에서 초기 UI 렌더링  
3. 팀원 / 활동 로그 입력 → 상태 변경 및 `autoSave()` 실행  
4. “요약 & PDF 생성” 선택 → `summary-engine.js` 집계 실행  
5. 대시보드 화면 표시  
6. PDF 요청 시 HTML을 iframe에 렌더링 후 출력

## 5) 기여도 점수 산정 규칙 

- 팀원별 총점은 다음과 같이 계산된다.  

  `scoreSum = Σ (각 로그에서 해당 팀원의 contributionScore)`

- 예) 로그 점수 5, 3, 4 → 총점 12
- 공동 작업 로그의 경우, 한 로그에 참여한 각 팀원에게 입력된 점수가 각각 누적된다.

> 즉, **활동 로그를 기준으로 참여자별 점수를 합산하여 기여도를 산정한다.**
